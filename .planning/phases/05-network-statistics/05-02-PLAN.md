**Goal:** Implement visual feedback for stale data when disconnected, preserving peer information so users can see the last known state.

## Task 01: Preserve Peers on Disconnect and Track lastUpdated

<read_first>
- EasyTierGUI/Services/ProcessViewModel.swift (NetworkRuntime class, lines 14-183)
- EasyTierGUI/Models/Models.swift (PeerInfo with lastUpdated property added in Plan 01)
</read_first>

<action>
In `EasyTierGUI/Services/ProcessViewModel.swift`, modify the `NetworkRuntime` class:

1. In the `service.$isRunning` sink (lines 43-57), modify the `else` branch (when `isRunning` is false) to preserve peers instead of removing them:

**Current code (lines 50-54):**
```swift
} else {
    self.status = .disconnected
    self.stopPeerPolling()
    self.peers.removeAll()
}
```

**Replace with:**
```swift
} else {
    self.status = .disconnected
    self.stopPeerPolling()
    // Keep last known peers but mark as stale (D-09)
    // Setting lastUpdated to nil will make isStale return false,
    // but we want to preserve the data for display
}
```

2. In the `fetchPeers()` method (lines 163-178), modify the callback to set `lastUpdated` timestamp:

**Current code (lines 169-177):**
```swift
service.fetchPeers(rpcPortalPort: port) { [weak self] newPeers in
    DispatchQueue.main.async {
        guard let self = self else { return }
        if self.peers != newPeers {
            self.peers = newPeers
            self.onStateChange?()
        }
    }
}
```

**Replace with:**
```swift
service.fetchPeers(rpcPortalPort: port) { [weak self] newPeers in
    DispatchQueue.main.async {
        guard let self = self else { return }
        let now = Date()
        if self.peers != newPeers {
            self.peers = newPeers.map { peer in
                var updatedPeer = peer
                updatedPeer.lastUpdated = now
                return updatedPeer
            }
            self.onStateChange?()
        }
    }
}
```

Do NOT change the `startPeerPolling()` or `stopPeerPolling()` methods - the 5-second polling interval remains unchanged (D-06, D-07).
</action>

<acceptance_criteria>
- `ProcessViewModel.swift` no longer has `self.peers.removeAll()` in the disconnect branch
- `ProcessViewModel.swift` `fetchPeers()` callback sets `lastUpdated = now` on each peer
- The comment `// Keep last known peers but mark as stale (D-09)` is present in the disconnect branch
- Project compiles without errors
</acceptance_criteria>
